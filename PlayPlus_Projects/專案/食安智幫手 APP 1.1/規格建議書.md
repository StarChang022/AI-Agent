### 2. 巡檢/設備表單截止提醒

*概述：* 當派案截止時間剩餘 50% 時通知負責人，剩餘 25% 時通知審核者（主管），以鈴鐺和 Email 雙重通知。

*功能規格：*

- *適用範圍：* 巡檢表單（InspectionForm）和設備表單（EquipmentForm）
- *計算基準：* 從「派發時間」到「截止時間」的總時長 - 剩餘比例 = (截止時間 - 現在) / (截止時間 - 派發時間)
- *觸發條件與對象：*

| 剩餘比例 | 通知對象 | 通知方式 |
  |---------|---------|---------|
  | ≤ 50% | 被指派人（assignee） | 鈴鐺 + Email |
  | ≤ 25% | 審核者（自訂層級） | 鈴鐺 + Email（含表單名稱及負責人資訊） |

- *通知頻率：* 每個閾值只通知一次
- *即使無人認領也須通知*，避免指派人忘記認領
- *退回重派時重新計算*提醒時間
- 25% 通知的審核者層級可自訂（需搭配 #7 多層簽核功能）

*技術範圍：*

- 後端：新增 Sidekiq 定期任務（建議每小時）、通知狀態追蹤欄位
- 新增通知類型：inspection_form_expiry_50、inspection_form_expiry_25、equipment_form_expiry_50、equipment_form_expiry_25
- 前端：通知顯示（現有機制）

*依賴關係：* 25% 通知層級設定依賴 #7 多層簽核功能

*預估工時：* 後端 4-5 天 / 前端 1 天（含設備表單）
*小計：5-6 天*

### 7. 多層簽核流程（巡檢 + 設備）

*概述：* 將現有的單層審核機制升級為可自訂層級的多層簽核流程，適用於巡檢表單和設備表單。

*功能規格：*

- *層級設定：* - 後台可自行設定簽核層級數量，*不限層級上限* - 每一層可指定多個審核者 - 同一層的審核規則：*任一人通過即可*
- *簽核流程：*
    
    填表人提交 → 第 1 層審核 → 第 2 層審核 → ... → 第 N 層審核 → 完成
      - 各層依序進行，前一層通過後才進入下一層
      - 任一層駁回 → 退回至*填表人（User）*，需重新提交
    
- *適用範圍：* - 巡檢表單（InspectionForm / InspectionFormApproval） - 設備表單（EquipmentForm / EquipmentFormApproval）
- *現有資料：* 不需要遷移

*技術範圍：*

- 後端： - InspectionSettingAssignment / EquipmentSettingAssignment 新增 approval_level 欄位 - InspectionFormApproval / EquipmentFormApproval 新增 level 欄位 - 重構審核狀態機（AASM）邏輯 - API 支援多層簽核操作
- 前端： - 後台 InspectionSetting / EquipmentSetting 編輯頁面：層級設定 UI - 表單簽核頁面：顯示當前層級與進度 - APP 端：簽核流程 UI 調整

*預估工時：* 後端 5-7 天 / 前端 4-5 天
*小計：9-12 天*
