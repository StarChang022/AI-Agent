## 工程開發規範與協調

1. 強型別不要使用 Any，採用 Class 包裝。
2. Mock Server 提供連結。
3. PlayPlus 提供 Web API 規格文件，提供後待 MiTAC 內部討論回饋。
4. API 多語系「簡中／繁中／英文／越南」。

## UX 調整

1. 通用。
    
    
    | SideBar 僅留下會議室相關功能。 |
    | --- |
    | 調整時段顯示方式，要有起訖時間，例如「08:00 - 08:30、08:30 - 09:00」。 |
2. 會議室清單
    
    
    | 左半部不需要特別留空間。一鍵快速預約重要性較高，往上移動；近期預約重要性較低，換個位置。 |
    | --- |
    | 日期 Switcher 中間改成「日期」。 |
    | 日期 Switcher 中間按鈕展開月曆。 |
    | 篩選條件由 MiTAC 後端提供。 |
    | 篩選條件都要可以複選。 |
    | 常用預約要顯示條件，旁邊加上 icon 顯示 Tooltips。 |
    | 篩選結果清單要盡可能顯示多筆結果，使用者需要掌握多間會議室的狀況。 |
    | 篩選結果清單的時段要顯示被誰借走，以便其他人可以協調時間。 |
3. 會議室清單（選擇預約時段）
    
    
    | Modal 需要可以左右切換會議室。**一般預約**時依照順序切換；**一鍵快速預約**時依照優先權高低切換。 |
    | --- |
    | 加強選擇方式，時段分成「早上／下午／晚上」，放上4顆功能按鈕「早上／下午／晚上／全選」。 |
4. 會議室清單 > 批次預約
    
    
    | 批次預約條件有「當週每天固定時段」及「當月固定每週幾與時段」這2種。 |
    | --- |
    | 介面大改，類似**會議室清單**，用批次預約條件篩選出符合條件的會議室，點擊會議室卡片後，跳出 Modal 確認預約所設定的條件。 |
