# 項目

請依照以下描述，為 MiTAC GSC 2.0 WebApp 建立高保真/線框圖 UI 設計。

# 需求
1. **介面風格**：現代且乾淨的企業級 SaaS 介接中心（例如淺灰色背景、無陰影的乾淨卡片、良好且易辨識的無襯線字體、明確有邏輯的 Tag 標籤色彩配置）。
2. **RWD 原則**：桌面端運用寬螢幕二/三欄版面顯示；針對手機端必須設計直式順暢堆疊卡片。使用 Modal 彈窗集中操作，不跳離主頁。
3. **語系支持**：導覽列需含語系切換下拉按鈕 (繁中, 簡中, 英文, 越南文)。

# 畫面1: 待處理簽核主清單頁面 (Main Workspace)
- **頭部 (Top Bar)**：左側放品牌 Logo「GSC」，右側放語系切換圖示與使用者頭像/姓名。
- **資訊與導覽區**：
  - 上方提供頁籤 (Tabs)：[簽核任務] / [歷史單據]。
  - 提供摘要卡圖 (Dashboard widgets)：顯示總數量、代理數量等。
- **過濾操作區 (Filter Bar)**：
  - 系統 (Dropdown：ERP/PLM/OA/ESCM)
  - 公司別 (Dropdown)
  - 其他標籤狀態 (下拉/Checkbox)
- **清單展示區**：
  - 以分頁 Data Table 或 List Card 展開。
  - 每列資料包含：[狀態標籤(如待簽核、正在聯繫)]、[文章標題/日期]、[系統與公司標籤]、[提交者名稱]、[代理標籤(含原負責人)]。

# 畫面2: 簽核詳情 Modal 視窗 (Detail Modal)
- 大型覆蓋彈出視窗（具備關閉 X 按鈕）。
- **主要內容區 (左右雙欄 或 上下排版)**：
  - 區塊 A (概要與詳細)：列出提交者、時間、系統名稱標籤，下方嵌入一大塊虛擬的動態 HTML Render 資訊區（可以白底加框線隔離）。
  - 區塊 B (簽核歷程)：右側或下方使用 Timeline 樣態，依序展示關卡編號、人員(或代理人)、狀態、完成時間、備註文字。
- **底部操作列 (Sticky Footer)**：
  - 右下角群組化顯示：[通過]、[通過並轉發]、[轉發]、[聯繫負責人]、[拒絕]。

# 畫面3: 疊加動作確認 Dialog
- **Dialog 視窗**：疊加於前一個 Modal 上。
- **文案**：顯示「確認動作」之警告與當前關卡。
- **輸入元件**：
  - AD 帳號搜尋框 (帶有搜索圖標，若為轉發功能時出現)。
  - 備註輸入框 Text Area。
- **底部行動按鈕 (Actions)**：
  - [取消] 
  - [確認送出]
  - [確認送出並續簽下一筆] (此為主軸重點按鈕，予以視覺突顯)。
