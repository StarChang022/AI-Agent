## 1. SSL / Cloudflare

用了 Cloudflare 並不代表你的 scorecard 就完全不會有 SSL/TLS 方面的問題。SecurityScorecard 在以下幾種情況下還是可能會標註偵測結果：

### CDN IP 輪轉：

Cloudflare 會頻繁更換 IP，這可能導致 SecurityScorecard 在短時間內從多個不同的 IP 看到你的憑證，並將每一次的偵測都視為獨立的問題。 [TLS Weak Cipher Suite]

### 特定 CDN 伺服器上的憑證過期：

有時候即使你的主要網站看起來沒問題，CDN 仍然會在某些特定的伺服器上提供舊版的憑證或加密套件。建議去檢查一下偵測結果中標註的特定 IP。 [TLS Weak Cipher Suite]

### 弱協定：

如果 Cloudflare 後端的任何端點（endpoint）仍然支援較弱的 TLS 協定，SecurityScorecard 就會標註它。如果因為業務需求真的無法更改設定，可以透過新增「公開議題評論」（Public Issue Comment）來提供背景資訊給正在查看你 scorecard 的人參考。 [SSL/TLS Weak Protocol]

### 建議：

請檢查 scorecard 中 SSL 偵測結果所標註的特定 IP 和連接埠 (port)，不要直接假設套了 Cloudflare 之後一切都會自動搞定。

## 2. 舊的或沒在使用的 IP 位址

如果該 IP 已經不屬於你的組織，你可以申請將它從你的「數位足跡」（Digital Footprint）中移除：

1. 前往 Digital Footprint 中的 IPs 標籤頁。
2. 選擇該 IP 並點擊「管理」（Manage）。
3. 檢查 SecurityScorecard 用來將此 IP 關聯到你的歸屬證據（attribution evidence）。
4. 選擇分類「IP 不應連結到此 Scorecard」並提供說明（以及任何證明文件）來證明這不是你的資產。 [Manage Assets]

### 注意更新時間：

即使移除申請獲得批准，該 IP 會顯示為「已移除」狀態，但在分數重新計算並發布之前的 2 到 3 天內，可能仍然會對分數產生影響。這是系統自動作業，無法手動調整。 [IP Removed but Score Still Impacted]

### 另外請注意：

如果該 IP 在還歸屬於你時就被偵測到問題，這些問題會根據 SecurityScorecard 的計分機制逐漸衰減（例如：惡意軟體偵測結果會在 30 天後過期）。如果你有證據證明在偵測到當時，該 IP 就已經不屬於你，你也可以透過「議題修復流程」（Issue Resolution Process）提交申請來解決。 [Misattribution]
