有個好消息，如果你已經不再使用那個 IP 了，有幾個明確的步驟可以處理這些檢測結果。以下是針對各個面向的處理方式：

### **為什麼這個 IP 還會出現？**

SecurityScorecard 的評分程序需要幾天的時間才能完成。如果這個 IP 是最近才輪換或釋出的，因為系統處理資料需要時間（從收集、處理到呈現大約要 2 天），它可能還會暫時出現在你的評分卡上。此外，DNS 歸屬的 TTL 是 1 天，這表示即便 DNS 紀錄已經不再指向該 IP，在短時間內它可能還是會被歸類在你的名下。**一旦歸屬的 TTL 過期，這些檢測結果就會自動消失。** [[**已輪換 IP 的常見問題**](https://support.securityscorecard.com/hc/en-us/articles/37925395410203-Findings-still-shown-on-already-rotated-IPs)]

### **你該怎麼做**

### **1. 在「數位足跡」中管理 IP**

最直接的方法就是透過「數位足跡（Digital Footprint）」對 IP 的歸屬提出異議：

1. 前往數位足跡中的 **IPs** 分頁。
2. 選擇該 IP 並點擊 **Manage（管理）**。
3. 查看 SecurityScorecard 用來將此 IP 連結到你網域的歸屬證據。
4. 如果你能證明該 IP 並未正確歸屬到你的網域，請選擇 **「IP should not be linked to this Scorecard（此 IP 不應連結至此評分卡）」** 類別。
5. 提供說明及任何佐證資料後提交。

一旦審核通過，該 IP 及其所有相關的檢測結果就會從你的評分卡中移除（大約在 72 小時內）。[[**管理資產**](https://support.securityscorecard.com/hc/en-us/articles/45103604591899-Manage-assets-or-request-their-removal)]

### **2. 針對 SSL/TLS 與加密套件（Cipher Suite）的檢測結果**

在歸屬問題解決前，如果該 IP 仍顯示有檢測結果，你也可以針對 SSL/TLS 和加密套件的發現提交「誤判（False Positive）」處理：

- 選擇該檢測結果 → 點選 **「Other resolutions（其他處置方案）」** → **「I cannot reproduce this issue and I think it's incorrect（我無法重現此問題，且認為這是錯誤的）」**。
- 系統會立即觸發重新掃描。如果之後沒再偵測到問題，系統會自動核准；如果問題依然存在，則會建立支援案件（Support Case）供進一步審查。[[**修正驗證**](https://support.securityscorecard.com/hc/en-us/articles/40395923414939-Remediation-validation-and-Support-ticket-creation)]

### **3. 針對 CVE／漏洞檢測結果**

對於高風險與中風險的漏洞檢測結果，如果你能證明以下情況，SecurityScorecard 可以將其移除：

- 你的公司從未使用過該 IP（歸屬錯誤），或
- 在發現檢測結果時，該 IP 尚不屬於你的組織。

你需要提交處置請求並附上佐證資料。[[**漏洞檢測結果**](https://support.securityscorecard.com/hc/en-us/articles/115002965683-Vulnerability-in-Last-Observation)]

---

**總結來說：** 先從「數位足跡」管理該 IP 並選擇「此 IP 不應連結至此評分卡」。這是最有效率的做法，因為一旦核准，就能一次處理掉所有相關的檢測結果。
