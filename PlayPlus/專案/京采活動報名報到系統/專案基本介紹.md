京采活動工程股份有限公司於 2016 年自行開發的活動報名報到系統，先從自己系統管理資料，再串接至客戶的活動網頁。輸入完相應的資料欄位後，從活動系統驗證資料及取得 QR Code 或報到編碼等，再完成報到手續。

系統當前架設於 AWS 雲端，使用10年尚未大改版，過程中經常 Debug 及修修補補，因為系統架構、系統介面太多陳舊，決定要直接大改版，不再疊床架屋。

# 重點概述

## 痛點

活動人數經常達到 3,000 人，依照目前活動流程，活動當天報到時容易造成系統負荷過大。

# 需求

系統架構重新開發、系統視覺 UI 重新設計，必須解決系統負荷問題，也必須優化 UX 讓初次使用者容易自己弄懂。

## 解決方案

重點在於解決系統負荷問題，除了重新優化程式碼，流程也需要調整，讓報到者分流。
