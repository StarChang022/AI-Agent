# 寫作格式

1. 文章最前面列出「## MetaTag」（不需要寫下標題「## MetaTag」）。
2. 每個段落一定要先寫下內文，才可以再接續子段落。例如 H2 段落必須要先寫內文，才可以再接續 H3 段落。
3. 文章內容依照「# 文章段落規則」格式撰寫。

## MetaTag

| 項目 | 值 |
| --- | --- |
| 標籤 | xxx |
| 列表簡述 | xxx |
| Title | xxx |
| Description | xxx |
| Keywords | xxx |
| URL | xxx |

- 標籤： 組成方式是從「### 標籤」裡面的「專案類型／產業／專案規模」各選1個來組成，例如「SAAS, 藝術產業, 中型專案」。
- 列表簡述： 簡短描述此頁面的內容，用於列表頁面。
- Title： 此頁面的標題。組成方式為「專案簡述 | 專案名稱」。
- Description： 此頁面的描述。
- Keywords： 此頁面的關鍵字。
- URL： 此頁面的網址結尾命名。

### 標籤

1. 專案類型： 品牌官網、電商平台、內容平台、企業內部系統、SAAS。
2. 產業： 餐飲業、金融服務、製造業、零售電商、電子業、藝術產業、生技醫療、不動產經營業。
3. 專案規模： 小型專案、中型專案、大型專案。

# 額外規定

1. 避免過度使用「不是…而是…」。
2. 減少使用「從…到…」。
3. 減少破折號。

## 背景與挑戰

此為 H2 段落，標題固定為「背景與挑戰」。先簡介客戶背景，再描述客戶遇到的挑戰。你當然不會知道當初客戶遇到什麼挑戰，如果我在[[暫存/說明.md]]裡面有提到，就從裡面發想；如果我沒有提到，請基於你的專業背景和過去的經驗，想像客戶可能遇到的問題。

### 關鍵痛點

此為 H3 段落，標題固定為「關鍵痛點」，屬於「## 背景與挑戰」的子段落。從客戶背景和挑戰中，找出客戶最在意的問題，並且要具體描述。

### 專案目標

此為 H3 段落，標題固定為「專案目標」，屬於「## 背景與挑戰」的子段落。描述客戶希望解決什麼問題，並且要具體描述。

## 解決問題的過程

此為 H2 段落，標題固定為「解決問題的過程」。你當然不會知道當初我們解決問題的過程，如果我在[[暫存/說明.md]]裡面有提到，就從裡面發想；如果我沒有提到，請基於你的專業背景和過去的經驗，想像客戶可能的解決過程。

請注意，解決過程必須要提到我們對於專案的看法、建議，以及我們如何透過「設計」來解決問題。

### 解決問題

此為 H3 段落，屬於「## 解決問題的過程」的子段落，用於提及「問題」與「解決方案」，所以標題要根據情境命名，例如「雙維度分類架構，幫助學員探索課程」或「主機成本的明確試算，化解對支出的不安」或「挑選影片串流服務」。

這個子段落會有複數個，我們解決越多問題，這個子段落就會寫越多。內容則根據標題的「問題」與「解決方案」來撰寫。
