| 題材 | 說明 |
| --- | --- |
| 品牌日常與幕後 | 團隊成員的日常做哪些事情、產品開發的幕後故事。例如「我們的設計師為了0.1公分的差距吵了一小時」。 |
| 互動與社群參與 | 轉發並評論粉絲的內容，分享提及粉絲，讓他們感到被重視。 |
| 價值提供與知識分享 | 分享在產業的「冷知識」或「小技巧」、快速解讀一則產業新聞，並分享「你的觀點」。 |
| 跟上時事與潮流 | 用品牌的角度去評論時下熱門的話題、迷因，重點在於「快速」與「有趣」，但是不要硬蹭。 |
| 產品服務的軟性連結 | 分享用戶使用產品後的好評與故事。或者預告新品及新活動，但用「聊天」的方式，而非「公告」，例如「我們最近在偷偷搞一個...大概是...」 |
| 知識教育的認知建構型 | 例如「什麼樣的企業適合自架網站，什麼時候該找顧問？」。 |
| 老闆喜歡的日常商業故事型 | 例如「數位轉型不是買工具，是改變習慣」、「我們遇過三種最常搞砸網站的決策模式」、「老闆花80萬做網站，卻想省1萬維運費，結果呢？」 |
| 工具推薦型 | 例如「推薦3個中小企業該用的數位工具」、「我們常用來協助流程自動化的工具清單」。 |
| 互動策略 | 固定每週「FAQ」，沒有題目就自己掰，用於鼓勵潛在客戶提出問題。 |
