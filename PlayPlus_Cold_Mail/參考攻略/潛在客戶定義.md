# 企業規模

- 員工人數： 約 5–100 人。
- 年營收： 數千萬至 1 億台幣。
- 組織結構： 扁平化，老闆或主管仍深度參與日常決策。

# 主要決策者

- 老闆／創辦人。
- 總經理。
- 營運主管。

# 決策者特徵

- 非技術背景。
- 以成本、風險、可控性為優先。
- 討厭「一次性大系統導入」。
- 容易被「MVP、先試再說」說服。

# 決策者心理

- 知道現況有問題。
- 不確定投資是否划算。
- 擔心做錯方向、做一半卡死。

# 典型描述用語

- 「現在就是先用 Excel 撐著」。
- 「之後有空再想系統」。
- 「之前有想過，但不知道怎麼開始」。
- 「找過軟體，看不懂就放著」。

# 常見現況

- 無內部 IT 團隊，或僅有外包維護。
- 網站多年未改版，或僅有形象用途。
- 系統多為單點工具，未整合。
- 訂單、報價、客戶資料以 Excel / Google Sheet 管理。
- 簽核、請款與流程，仍依靠 Email、LINE 或紙本。
- 資料重複填寫，版本混亂。
- 無法即時掌握營運狀況。
- 人員流動導致流程失控。
- 公司規模擴大，Excel 開始撐不住。
- 老闆想「看即時數據」卻看不到。
- 新接大客戶，流程要求變嚴格。
- 政府稽核、ISO、資安要求出現。
- 既有系統太舊，維護成本過高。
