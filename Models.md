# 快速比較

Gemini 3.1 Pro (High)： 處理複雜的金融報表分析、超大型程式碼庫、極高精確度的科學推理。
Gemini 3 Pro (Low)： 長文本總結、內容創作、具有上下文記憶的日常助手。
Gemini 3 Flash： 簡單小任務。
Claude Sonnet 4.6 (Thinking)： 程式開發。
Claude Opus 4.6 (Thinking)： 擬定策略或撰寫合約。
GPT-OSS 120B (Medium)： 數據分析。

---

# 詳細介紹

## Gemini 3.1 Pro (High)

啟用最高等級的 thinking_level。這款模型會花費更多預算進行內部的「鏈式思考」（Chain-of-Thought），專門對抗邏輯謬誤。

**處理複雜的金融報表分析、超大型程式碼庫（Repo）的系統架構審查、或需要極高精確度的科學推理。**

## Gemini 3 Pro (Low)

縮減了推理思考的時間與權杖消耗。它保有 3.1 Pro 的廣博知識庫，但在回答簡單問題時反應極快，不會「過度思考」。

**日常的長文本總結、進階的內容創作、或是作為具有上下文記憶的日常助手。**

## Gemini 3 Flash

速度與成本的極致優化。雖然是「Flash」輕量版，但它在 3.0 版本後獲得了飛躍性的邏輯提升，特別是「Agentic（代理人）」能力。

**需要即時反應的語音對話（Live Mode）、大規模自動化處理（RPA）、以及高頻率的小任務。**

## Claude Sonnet 4.6 (Thinking)

被譽為 2026 年最強的「程式開發模型」。在開啟 Thinking 模式後，它在軟體工程（Coding）和系統設計上的表現甚至超越了許多旗艦模型，且價格更具競爭力。

**複雜 Bug 調試、自動化開發代理、長連貫邏輯規劃。**

## Claude Opus 4.6 (Thinking)

Anthropic 的最高階旗艦，擁有最強的「人文感」與「策略推理」。Opus 4.6 在處理具有模糊性或道德爭議的問題時，思考深度最深，輸出的 Token 上限（Output Limit）也最高（可達 128k）。

**法律合約深度審查、策略性商業規劃、高難度創意寫作。**

## GPT-OSS 120B (Medium)

採用了 Mixture-of-Experts (MoE) 架構，雖然總參數高達 117B，但每次啟動僅約 5B 參數。其 "Medium" 代表的是其「推理努力值（Reasoning Effort）」被設定在平衡點。

它是目前少數支援「完整思考鏈可視化」的強大模型，且支援本地部署（使用 H100 等級顯卡可單卡運行），安全性與隱私控管極佳。

**企業內部數據分析、需要透明化思考路徑的研發環境、以及對數據隱私要求極高的本地開發。**
