# Contributing

Contributions are welcome! Please read the following guidelines before submitting a pull request.

## Adding a Resource

- Search existing entries to avoid duplicates.
- Make sure the resource is genuinely useful and relevant to Generative Engine Optimization.
- Use the following format: `- [Name](url) - Description.`
- The description should start with an uppercase letter and end with a period.
- Add new entries at the bottom of the relevant section.

## Pull Request Process

1. Fork this repository.
2. Create a new branch for your changes.
3. Make your changes and commit with a clear message.
4. Open a pull request describing what you added and why.

## Quality Standards

- Links must be accessible and point to high-quality, maintained resources.
- No promotional or spammy content.
- Descriptions should be objective and concise.

Thank you for helping make this list better!
