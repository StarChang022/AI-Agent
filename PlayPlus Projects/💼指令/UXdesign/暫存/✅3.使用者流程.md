# 項目

- **流程名稱**：日常集中連做簽核 (批次處理)
  - **使用者角色**：高階主管 (David)
  - **使用者目標**：快速看完所有系統待簽單並一次處理。
  - **觸發情境**：打開 GSC 2.0 WebApp。
  - **前置條件**：已登入且有待審單。
  - **主要操作步驟**：
    1. 進入首頁總清單，查看資訊摘要區得知數量。
    2. 點擊第一筆進入「簽核詳情 Modal」。
    3. 檢視後點擊「通過」按鈕。
    4. 彈出「通過 Modal」，點選「確認通過並續簽下一筆」。
    5. Modal 自動載入下一筆單據。重複至處理完。
  - **例外情況**：若遇到疑慮單據，可點「拒絕」，或點右上方取消退出連續流程。
  - **終點**：看完所需單據，清單更新。

- **流程名稱**：使用標籤篩選處理代理並轉發
  - **使用者角色**：代理人/特助 (Linda)
  - **使用者目標**：將代簽單挑選並轉發給其他單位。
  - **觸發情境**：主管出差授權代簽。
  - **前置條件**：擁有代理權限，且清單有代理單據。
  - **主要操作步驟**：
    1. 於清單上方的篩選區設定下拉為「代理」。
    2. 點開一筆帶有「代理 / OOO(原負責人)」標籤的單據 Modal。
    3. 點擊「轉發」。
    4. 於「轉發 Modal」中搜尋法務人員的 AD 帳號並選定。
    5. 填寫備註並點擊「確認轉發」。
  - **例外情況**：AD 帳號搜尋不到，需提醒或允許模糊查詢。
  - **終點**：單據發送給他人並從自己的待辦中移除。

- **流程名稱**：單據存疑之聯繫討論 (Face to Face)
  - **使用者角色**：財務主管 (Sarah)
  - **使用者目標**：決定通過前，向提交者釐清金額異常。
  - **觸發情境**：歷程資訊或附件數據有誤。
  - **前置條件**：正閱讀簽核詳情 Modal 時。
  - **主要操作步驟**：
    1. 於簽核詳情 Modal 中點擊「聯繫負責人」按鈕。
    2. 顯示對話框告知將發送信件且單據將進入等待狀態。
    3. 點選「確認聯繫」。
    4. Modal 自動關閉，返回任務清單。
  - **例外情況**：誤點可按取消。
  - **終點**：清單中該單據被掛上「正在聯繫負責人」狀態。

- **流程名稱**：多語系下的歷史單據調閱
  - **使用者角色**：海外廠長 (Jason)
  - **使用者目標**：用越文介面查詢舊的處理紀錄。
  - **觸發情境**：遇到問題需要翻舊紀錄。
  - **前置條件**：登入完成。
  - **主要操作步驟**：
    1. 點擊右上角語系切換器，切為「越南文」。
    2. 點選清單區的「歷史單據」頁籤。
    3. 開啟歷史列表，利用年份/系統別篩選後，點開檢視內頁(或 Modal)。
  - **例外情況**：對應語系下若無系統資料，需有佔位提示。
  - **終點**：看完單據，關閉 Modal。
