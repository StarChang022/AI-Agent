# 項目

- **Persona 1: 行動辦公的高階主管 (David)**
  - **基本資料**：David, 52歲, 總經理/集團事業群主管, 製造與科技業。常使用 iPhone 瀏覽, 數位熟悉度中上。
  - **背景描述**：每天有多場內外會議，需處理集團各子公司(MIC等)的各類重大簽核(ERP, ESCM等)。
  - **引言**：「幫我把所有要簽的集中好，我只要幾分鐘快速滑過去確認就好，不要浪費我切換系統的時間。」
  - **主要目標**：用最短時間在手機上完成積壓的簽核，在意效率與跨系統整合度。
  - **痛點與困擾**：不想點進舊系統層層資料夾裡找單子，系統換來換去很麻煩。
  - **行為模式**：常在車程或會議空檔快速批次看單子。
  - **動機與在意點**：要求即時性，介面必須大且清楚。
  - **使用情境**：在計程車上、高鐵上用手機查看 GSC。
  - **設計啟示**：手機版需極度友善、字體大且按鈕清晰，能夠「通過並續簽下一筆」。

- **Persona 2: 嚴謹細心的部門主管 (Sarah)**
  - **基本資料**：Sarah, 45歲, 財務長/財務部協理, 數位熟悉度高, 筆電與桌機為主。
  - **背景描述**：負責多個財務系統審核，需仔細看過單據的 HTML Detail Info，注重簽核歷史歷程跟留言。
  - **引言**：「我必須很清楚這張單前面誰簽過、意見是什麼，才能決定要不要放行。」
  - **主要目標**：正確無誤且有憑有據地進行簽核，注重資訊全面性與防呆。
  - **痛點與困擾**：歷程資訊難找，有時候無法看到前一關的備註；舊系統 iFrame 太小難排版。
  - **行為模式**：仔細看歷程，若有疑問會「轉發」或首選「聯繫負責人」。
  - **動機與在意點**：重視正確性、權責劃分。
  - **使用情境**：在辦公室使用雙螢幕進行仔細核對。
  - **設計啟示**：Detail Info 展開要夠大，歷程紀錄的時間軸排版要清楚明瞭。

- **Persona 3: 常需找人代簽與調整流程的業務主管 (Kevin)**
  - **基本資料**：Kevin, 38歲, 業務總監, 數位熟悉度非常高, 平板和筆電並用。
  - **背景描述**：常在國外出差拜訪客戶，需要請特助代簽，也常臨時新增關卡（通過並轉發給法務等單位）。
  - **引言**：「這張單我同意，但我需要法務再看一下，幫我加派給法務單位。」
  - **主要目標**：彈性調度簽核流程，順暢進行轉發。
  - **痛點與困擾**：舊系統難以臨時加人或是指派人名不滑順。
  - **行為模式**：快速瀏覽後決定是否要別人背書。
  - **動機與在意點**：協作效率、即時轉發不卡單。
  - **使用情境**：在機場候機時用平板操作連網。
  - **設計啟示**：「通過並轉發」的 AD 帳號搜尋欄需具備智慧下拉選單，介面提示明顯。

- **Persona 4: 協助過濾單據的代理簽核人 (Linda)**
  - **基本資料**：Linda, 32歲, 總監特助, 數位熟悉度高, 桌機為主。
  - **背景描述**：幫主管（如 Kevin）代簽日常庶務表單。
  - **引言**：「我要知道這張單我是代理誰簽的，免得簽錯人的單據。」
  - **主要目標**：明確區分自己的單與長官交辦的單。
  - **痛點與困擾**：清單混雜，無法一眼找出是代替誰簽核的項目。
  - **行為模式**：先用篩選器挑出特定長官的單子，統一處理。
  - **動機與在意點**：不越權、標籤明確安全。
  - **使用情境**：早晨進辦公室後第一件事批次處理雜項。
  - **設計啟示**：標籤系統需要清楚呈現「代理」及原負責人名稱，且篩選器應有相應選項。

- **Persona 5: 需追蹤進度與歷史的基層主管 (Jason)**
  - **基本資料**：Jason, 35歲, 海外廠長/產線主任, 越南籍, 手機及桌機皆會用。
  - **背景描述**：簽核異常通報，並常需要看歷史紀錄確認處理狀況。
  - **引言**：「上次機台怎麼修的？我切換越南文介面查一下歷史簽核單。」
  - **主要目標**：快速找到歷史表單，跨語系無障礙。
  - **痛點與困擾**：系統若只有中文會看不懂，歷史紀錄難找。
  - **行為模式**：頻繁切換語系，依賴搜尋功能。
  - **動機與在意點**：語言在地化。
  - **使用情境**：工廠透過手機切換越南文操作。
  - **設計啟示**：完整支援四語言(中繁, 中簡, 英, 越)語系切換，歷史單據區需要易用。
