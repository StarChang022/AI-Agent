# 項目

- **專案背景**：
MiTAC 集團內部有多個系統（ERP、PLM、OA、ESCM 等）需要簽核，原有主管須逐個系統開啟簽核，造成極大不便。因此開發了集中式的 GSC (Global Sign-off Center) 1.0 簽核系統供主管使用。但因 1.0 版本未經過專業設計師設計，操作上仍有許多不便，例如：傳統的左側資料夾導覽不易閱讀、未能整合所有待簽核事項在同一清單頁、iFrame嵌入顯示模式體驗不佳。為改善使用者體驗，現啟動 GSC 2.0 改版專案。

- **專案目標**：
重新設計 GSC 2.0 簽核中心介面。簡化並集中所有系統（含未來擴充）的待辦簽核至同一清單，提供進階的標籤篩選與排序功能，優化行動版（RWD）與多語系（繁中、簡中、英文、越文）支援。優化簽核彈窗的操作流程，讓主管能有效率地在不離開清單頁面的前提下，完成「通過、拒絕、轉發、通過並轉發、聯繫負責人」等操作。

- **專案範圍**：
  - **要做什麼**：
    - 所有簽核事項合併於同一清單頁面，運用標籤與篩選器管理。
    - 簽核詳情及操作改為 Modal 彈出視窗（包含通過、拒絕、轉發、通過並轉發、聯繫負責人）。
    - 提供歷程進度顯示及代理人資訊標示。
    - 提供歷史單據查詢頁面。
    - 支援電腦版與手機版（RWD）設計。
    - 支援多語系架構設計。
    - 大部分動作改為 Modal 操作，減少頁面跳轉。
  - **不做什麼**：
    - 原系統的 Detail Info 內容排版（由各系統端回傳動態 HTML 渲染）。
    - GSC 系統以外的其餘 MiTAC 原系統內部功能開發。

- **功能需求**：
  1. **跨系統簽核任務清單**：
     - **功能名稱**：跨系統簽核任務清單
     - **說明**：集中展示所有系統或子公司的待辦簽核清單。
     - **操作對象**：具備簽核權限的主管、代理簽核人。
     - **前置條件**：登入系統且有待處理之簽核項目。
     - **操作流程**：使用者一進系統即看見清單，可由上方資訊摘要區查看總數、使用篩選器篩選（依系統/公司別）。
     - **欄位規則**：提供狀態、標題、日期、提交者、系統/公司別標籤、代理/原負責人。具備頁碼分頁。
  2. **簽核詳情 Modal**：
     - **功能名稱**：簽核詳情 Modal
     - **說明**：查看單據詳細內容及簽核歷程，並提供動作按鈕。
     - **操作對象**：具備簽核權限的主管、代理簽核人。
     - **前置條件**：點擊清單中的簽核項目。
     - **操作流程**：點擊列表開啟 Modal ➜ 查看 HTML 渲染的單據詳情及歷程 ➜ 點擊所需動作按鈕。
     - **欄位規則**：包含重要資訊（提交者等）、詳細資料（動態 HTML）、簽核歷程（關卡、簽核人字與備註），以及動態動作按鈕。
  3. **動態簽核動作 Modal（通過/拒絕/轉發/聯絡）**：
     - **功能名稱**：各項簽核動作再確認 Modal
     - **說明**：輸入備註或轉發帳號。
     - **操作對象**：正在檢視單據且具權限的主管。
     - **前置條件**：於簽核詳情點擊相應的動作按鈕。
     - **操作流程**：點擊動作按鈕 ➜ 填寫對應欄位 ➜ 選擇確認送出 或 確認送出並續簽下一筆。
     - **欄位規則**：拒絕原因必填；轉發需輸入 AD 帳號必填；備註選填。
  4. **歷史單據查詢**：
     - **功能名稱**：歷史紀錄瀏覽區
     - **說明**：查閱過去已處理的歷史紀錄。
     - **操作對象**：所有主管。
     - **前置條件**：點擊切換至歷史單據。
     - **操作流程**：點選歷史單據列表開啟檢視。

- **驗收標準**：
  - 清單可跨4大系統集中顯示，且可正確透過下拉式篩選器篩選項目。
  - 所有簽核詳情及操作彈窗（通過、拒絕、轉發等）皆於當前頁面以 Modal 正確開啟且支援「確認並續簽下一筆」。
  - 任務若為「代理」，應顯示「代理」及原負責人名稱。
  - 介面在手機版（Mobile）具可讀性並可順暢簽核（RWD）。
  - 送出「聯繫負責人」後，該單據狀態需標示「正在聯繫負責人」。
