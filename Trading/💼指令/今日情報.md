# 寫作格式規則

1. 報告開頭要獨立寫出「今天」的日期作為報告撰寫日期。
2. 「# 個股」與「# 大盤」的情報，必須獨立寫出，不要混在一起。所以報告內應該是先列出「# 大盤」段落（內容包含「## 報告重點」），再列出「# 個股」段落。
3. 承上，大盤段落的格式大致為先寫「# 大盤」，再寫「## 關鍵驅動因素」及「## 資金流向觀測」等...依此類推。
4. 承上，個股段落的格式大致為先寫「# 個股」，再寫「## 股票名稱+代號」，有幾檔股票就寫幾列，每列的格式同樣為「## 股票名稱+代號」。
5. 「## 總結」應該是寫在大盤段落的最後，為大盤做總結。

# 額外規範

1. 禁止使用「---」分隔線。
2. 每個段落都要從 ## 這種 <h2> 開始使用，也就是說大段落為 <h2>，裡面如果還有子段落，則依序為 <h3> 和 <h4>。
3. 你自己調查就好，不准叫我開啟瀏覽器！
